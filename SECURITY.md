# Security Policy

Please do not report security vulnerabilities through public GitHub issues.

This repository is public and intended for marketing, roadmap visibility and community issue tracking. It should never contain private application source code, `.env` files, tokens, customer data, crash dumps with credentials, signing keys, payment keys or internal deployment files.

## Reporting A Vulnerability

Email security reports to:

**support@termiprotocol.com**

Include:

- affected platform: macOS, Windows, Linux or website
- app version if available
- steps to reproduce
- expected and actual result
- logs or screenshots with secrets removed

We will review reports and follow up with next steps.

## Public Issue Safety

For public bugs and feature requests, remove:

- API keys and provider tokens
- Supabase, Polar, GitHub or payment credentials
- private repository names if sensitive
- file paths that reveal private customer or company data
- proprietary source code snippets
